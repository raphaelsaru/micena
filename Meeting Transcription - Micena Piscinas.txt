# Resumo da Reunião

## Objetivo Geral

Desenvolver um sistema para gerenciar clientes, rotas, serviços e o controle financeiro relacionado, utilizando inteligência artificial para auxiliar na criação da estrutura e código.

## Funcionalidades Principais do Sistema

### 1\. Cadastro e Gerenciamento de Clientes

*   Tela para CRUD (criar, ler, atualizar, deletar) clientes.
    
*   Campos essenciais: CPF/CNPJ, e-mail, endereço, CEP, telefone, chave Pix.
    
*   Identificação de clientes com pagamento recorrente ou não.
    
*   Controle dos pagamentos: meses pagos, meses em atraso e status do pagamento do mês vigente.
    
*   Possibilidade de anexar comprovantes de pagamento, que, ao serem adicionados, marcam o pagamento automaticamente.
    

### 2\. Serviços Prestados aos Clientes

*   Tela para cadastro e gerenciamento dos serviços prestados a cada cliente.
    
*   Registro detalhado dos serviços (exemplos: troca de areia, equipamentos, capa da piscina), com campo para especificação dos equipamentos.
    
*   Cadastro da data do serviço e data para próximo atendimento.
    
*   Emissão de ordem de serviço ou recibo, com possibilidade de enviar ao cliente.
    
*   Integração com Google Agenda para criar eventos de lembrete sobre serviços futuros ou recorrentes.
    

### 3\. Controle Financeiro

*   Visão focada no financeiro, mostrando situação dos pagamentos dos clientes.
    
*   Gestão dos clientes mensalistas com lista dos meses pagos e em aberto.
    
*   Inclusão de comprovantes de pagamento.
    
*   Relacionamento constante com o cadastro dos clientes por meio de IDs.
    

### 4\. Gerenciamento de Rotas e Agenda

*   Menu com abas horizontais para navegação entre os dias úteis (segunda a sexta).
    
*   Definição de número máximo de clientes atendidos por dia.
    
*   Distribuição dos clientes em colunas para melhor visualização (5 clientes por coluna, após 5 inicia nova coluna).
    
*   Ordenação decrescente automática dos clientes, com possibilidade de reordenação via drag and drop.
    
*   Atualização em tempo real da ordenação ao modificar o número máximo de clientes ou reordenar manualmente.
    
*   Inserção de clientes em posições específicas, com reordenação automática para evitar duplicidade na numeração.
    

### 5\. Interface do Usuário e Estilo Visual

*   Uso do framework NextJS.
    
*   Banco de dados próprio para persistência de dados.
    
*   Interface moderna, amigável, responsiva para celular e com estilo CSS atualizado.
    
*   Possibilidade de customizar fontes, tamanhos e cores para relatórios e impressões, incluindo bordas e números.
    

### 6\. Sistema de Autenticação

*   Tela de login com autenticação via Supabase.
    
*   Apenas um login para o cliente (tipo admin).
    
*   A equipe de desenvolvimento terá acesso total ao código e hospedagem, mas o cliente não terá acesso ao código base nem ao ambiente de hospedagem.
    
*   Possível acesso separado para equipe de manutenção/supervisão, se necessário.
    

## Relatórios e Impressão

*   Sistema permitirá imprimir as telas, especialmente a página de rotas, com opções de customização do estilo visual.
    
*   Necessidade de gerar documentos semelhantes aos orçamentos PDF que o cliente já utiliza.
    
*   Tela para consulta do histórico de serviços de cada cliente.
    

## Observações Técnicas e Estratégicas

*   Uso de inteligência artificial para:
    
    *   Criar a estrutura do banco de dados (tabelas, colunas, relacionamentos).
        
    *   Gerar o código do sistema via agentes (exemplo: GitHub Copilot ou similar).
        
*   Organização do desenvolvimento com roadmap (plano de implementação) e instruções detalhadas para garantir qualidade e eficiência.
    
*   A equipe não é de programadores especializados, por isso a dependência de ferramentas e IA para implementação.
